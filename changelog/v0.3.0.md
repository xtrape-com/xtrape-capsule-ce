# v0.3.0 Changelog

Status: historical release-train notes for the v0.3 OpHub-focused update.

## Theme

OpHub mode and Opstage UX support for multi-service local runtime management.

## Added

- `mode: "ophub"` Agent registration support for OpHub-managed multi-service reporting.
- Backend registration/reporting compatibility for OpHub-managed Capsule Services.
- Acceptance of v0.3 contract payloads used by OpHub/demo reports, while keeping CE persistence focused on existing service/action/config/health tables.
- UI/backend compatibility updates for OpHub-managed Capsule Services.

## Changed

- Updated dependencies to v0.3 contracts and SDK packages.
- Registration schema boundary clarified so CE can accept OpHub registrations without an embedded service payload.
- CI validation aligned with current typecheck/build commands.

## Compatibility

- Pairs with contracts-node `0.3.x`, Agent SDK `0.3.x`, OpHub Go v0.3 runtime work, demo adapter mode, docs, and site.
- First-class CE persistence/rendering for capability/event catalogs remained limited; contracts and demo carried most metadata shape work in v0.3.
