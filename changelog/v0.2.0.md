# v0.2.0 Changelog

Status: historical release notes for the v0.2 Public Preview update.

## Theme

Operational correctness, status semantics, metrics, and UI maintainability.

## Added

- Effective status semantics for stale/offline Capsule Services and Agents.
- Command failure fields surfaced directly on command rows (`errorCode`, `errorMessage`, `durationMs`).
- Richer `/api/admin/metrics`, system health/version endpoints, OCI image metadata, diagnostics, and Docker tag documentation.
- Heartbeat audit sampling and token `lastUsedAt` write throttling.
- ADR/documentation for ephemeral action secrets and operational behavior.

## Changed

- Tightened command lifecycle races, retry restrictions, and Agent token rotation on re-register.
- Refactored the UI from a large `App.tsx` into page/lib components for maintainability.
- Hardened secret redaction in audit writes and command result handling.
- Docker image publishing adjusted for v0.2 release behavior.

## Compatibility

- Pairs with contracts-node and Agent SDK `0.2.x`.
