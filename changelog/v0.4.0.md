# v0.4.0 Changelog Draft

Status: draft until the v0.4 release train is tagged and accepted.

## Theme

Capsule Bus Experimental.

## Added

- Feature-flagged Capsule Bus backend surface guarded by `OPSTAGE_CAPSULE_BUS_ENABLED=false` by default.
- Bus persistence tables for events and routes, with `bev_` and `brt_` ID prefixes.
- Admin APIs for Bus route CRUD, Bus event listing, and Bus audit listing.
- Agent event ingestion API at `/api/agents/{agentId}/bus/events`.
- In-process event-to-command routing for `ACTION_EXECUTE` command creation, DRY_RUN routes, source-service ownership checks, target/action validation, and audit events.
- Admin UI Capsule Bus page with route management, event view, and Bus audit tab.
- Integration tests covering disabled feature behavior, auth ordering, route CRUD, source validation, redaction, event ingestion, command creation, and audit visibility.

## Security and governance

- Admin Bus APIs require session auth and operator permission for mutations before feature-flag state is exposed.
- Agent event publishing requires Agent token auth and ownership of the source service code.
- Bus payload and metadata are redacted before persistence and command payload creation.

## Notes

- Capsule Bus APIs are experimental and not stable before v1.0.
- No external broker, queue, service mesh, load balancing, or marketplace workflow is included in v0.4.
